# DNASPEC 技能快速参考指南

## 📋 技能概览

DNASPEC 提供4个符合 agentskills.io 标准的核心技能：

| 技能 | 功能 | 使用时机 |
|-----|------|---------|
| **dnaspec-architect** | 系统架构设计 | 架构设计、系统规划、微服务 |
| **dnaspec-context-analysis** | 上下文质量分析 | 评估内容质量、识别问题 |
| **dnaspec-context-optimization** | 上下文质量优化 | 改进内容、提升质量 |
| **dnaspec-cognitive-template** | 结构化认知模板 | 分步推理、示例引导、验证 |

---

## 🎯 dnaspec-architect

**核心功能**：复杂项目的分层架构设计、任务分解、智能体化和约束生成

### 何时使用

- ✅ "设计电商平台的架构"
- ✅ "规划微服务系统"
- ✅ "创建智能体系统"
- ✅ "系统架构评估"

### 何时不使用

- ❌ 具体代码实现
- ❌ 算法优化
- ❌ 调试代码错误

### 典型输出

```
1. 需求分析报告
2. 架构模式选择
3. 系统架构图
4. 服务划分方案
5. 技术栈建议
6. 实施计划
```

### 示例调用

```
"请使用 dnaspec-architect 设计一个支持10万并发的电商平台架构"
```

## 🔍 dnaspec-context-analysis

**核心功能**：五维质量评估（清晰度、相关性、完整性、一致性、效率）

### 何时使用

- ✅ "分析这段对话的质量"
- ✅ "评估文档的有效性"
- ✅ "检查内容是否完整"
- ✅ "识别沟通中的问题"

### 五维评分

| 维度 | 评分范围 | 评估重点 |
|-----|---------|---------|
| **清晰度** | 0.0-1.0 | 语言精确性、逻辑清晰度 |
| **相关性** | 0.0-1.0 | 主题契合度、目标一致性 |
| **完整性** | 0.0-1.0 | 信息覆盖度、要素完备性 |
| **一致性** | 0.0-1.0 | 逻辑连贯性、术语统一性 |
| **效率** | 0.0-1.0 | 信息密度、简洁程度 |

### 典型输出

```json
{
  "clarity": 0.82,
  "relevance": 0.75,
  "completeness": 0.68,
  "consistency": 0.80,
  "efficiency": 0.70,
  "issues": ["缺少关键细节"],
  "suggestions": ["补充实施步骤"]
}
```

### 示例调用

```
"使用 dnaspec-context-analysis 分析这段需求文档的质量"
```

## ✨ dnaspec-context-optimization

**核心功能**：基于特定目标优化上下文质量

### 何时使用

- ✅ "优化这段文档"
- ✅ "提高表达的清晰度"
- ✅ "精简冗余内容"
- ✅ "统一术语表述"

### 五大优化目标

1. **清晰度增强**
   - 精确语言替换模糊术语
   - 改善逻辑结构
   - 添加必要解释

2. **相关性改进**
   - 删除离题内容
   - 加强主题聚焦
   - 补充相关细节

3. **完整性增强**
   - 填补信息空缺
   - 添加背景说明
   - 补充必要细节

4. **简洁度改进**
   - 删除冗余内容
   - 精简表达
   - 消除重复

5. **一致性增强**
   - 统一术语
   - 解决矛盾
   - 确保流畅

### 典型输出

```json
{
  "before": {"clarity": 0.65},
  "after": {"clarity": 0.85},
  "improvements": [
    "重写模糊的技术描述",
    "删除重复内容"
  ]
}
```

### 示例调用

```
"使用 dnaspec-context-optimization 优化这段API文档，提升清晰度和简洁度"
```

## 🧠 dnaspec-cognitive-template

**核心功能**：5种结构化认知模板，提供框架化思维支持

### 何时使用

- ✅ "分步思考这个问题"
- ✅ "举例说明"
- ✅ "验证这个方案"
- ✅ "从架构师角度分析"
- ✅ "深入理解这个概念"

### 五大认知模板

#### 1. 思维链 (Chain of Thought)
**用途**：分步推理、复杂问题求解
```
📋 问题理解
🔍 步骤分解
💭 中间推理
✅ 验证检查
📝 最终答案
```

#### 2. 少样本学习 (Few-Shot Learning)
**用途**：示例引导、模式识别
```
📚 示例学习 (1, 2, 3)
🎯 应用规律
🔍 验证结果
```

#### 3. 验证框架 (Verification)
**用途**：质量检查、结果确认
```
🎯 初始判断
🔍 逻辑一致性检查
✅ 事实准确性检查
📋 完整性检查
🔮 最终确认
```

#### 4. 角色扮演 (Role Playing)
**用途**：专业视角分析
```
🎭 角色定义 (如: 资深架构师)
🔍 角色理解
💡 角色分析
🎯 角色建议
🤝 角色决策
```

#### 5. 理解框架 (Understanding)
**用途**：深度理解、全面分析
```
🎯 核心目标
📦 关键要素
⚙️ 约束条件
✅ 成功标准
⚠️ 潜在风险
📊 优先级分析
```

### 示例调用

```
"使用 dnaspec-cognitive-template 的思维链模板分析这个架构设计"
"使用角色扮演模板，从资深架构师角度评估这个方案"
```

## 🔄 技能组合使用

### 场景1: 架构设计 + 分析

```
1. 使用 dnaspec-architect 设计架构
2. 使用 dnaspec-context-analysis 评估设计文档质量
3. 使用 dnaspec-context-optimization 优化文档
```

### 场景2: 问题解决 + 认知模板

```
1. 使用 dnaspec-cognitive-template 的思维链分析问题
2. 使用 dnaspec-context-analysis 评估解决方案
3. 使用验证框架验证最终方案
```

### 场景3: 文档优化全流程

```
1. 使用 dnaspec-context-analysis 识别问题
2. 使用 dnaspec-context-optimization 优化内容
3. 使用 dnaspec-context-analysis 再次评估
4. 使用 dnaspec-cognitive-template 的理解框架确保完整性
```

## 📊 技能对比

| 维度 | architect | context-analysis | context-optimization | cognitive-template |
|-----|-----------|------------------|---------------------|-------------------|
| **主要输出** | 架构设计文档 | 质量评估报告 | 优化后内容 | 结构化分析 |
| **输入要求** | 项目需求 | 待分析内容 | 待优化内容 | 问题/任务 |
| **处理时间** | 长（复杂分析） | 中 | 中 | 可长可短 |
| **输出格式** | Markdown文档 | JSON + 报告 | 对比 + JSON | 结构化文本 |
| **可量化性** | 低 | 高（评分） | 中 | 低 |

## 💡 最佳实践

### 1. 明确目标

在使用技能前，明确你的目标：
- 需要架构设计？ → dnaspec-architect
- 需要评估质量？ → dnaspec-context-analysis
- 需要改进内容？ → dnaspec-context-optimization
- 需要结构化思维？ → dnaspec-cognitive-template

### 2. 提供充分上下文

给技能足够的上下文信息：
- ✅ "设计一个电商平台，技术栈Node.js，支持10万并发"
- ❌ "设计一个系统"

### 3. 关注输出质量

检查技能输出是否：
- ✅ 回答了你的问题
- ✅ 提供了具体的建议
- ✅ 包含了可执行的方案
- ✅ 有清晰的格式和结构

### 4. 迭代优化

技能支持迭代：
```
第1次: 使用 context-analysis 发现问题
第2次: 使用 context-optimization 优化
第3次: 再次使用 context-analysis 验证改进
```

### 5. 结合使用

不要局限于单一技能：
```
architect (设计) + context-analysis (评估) + context-optimization (优化)
```

## 🎓 学习路径

### 初级用户

1. 先熟悉 **dnaspec-context-analysis**
   - 学习五维质量评估
   - 理解评分标准

2. 再学习 **dnaspec-cognitive-template**
   - 掌握思维链模板
   - 学会结构化思考

### 中级用户

3. 学习 **dnaspec-context-optimization**
   - 理解优化目标
   - 掌握优化策略

4. 组合使用技能
   - analysis + optimization
   - cognitive-template + analysis

### 高级用户

5. 深入 **dnaspec-architect**
   - 理解架构设计流程
   - 掌握系统化方法

6. 综合运用所有技能
   - 完整项目生命周期
   - 多技能协作
