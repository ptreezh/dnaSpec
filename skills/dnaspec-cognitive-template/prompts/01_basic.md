# DNASPEC Cognitive Template - 基础应用

## 基础模板类型

### 1. 理解模板 (Understanding Template)

**用途**：快速理解问题和需求

```yaml
understanding_template:
  structure:
    1. 识别核心问题
    2. 分析约束条件
    3. 理解目标期望
    4. 识别关键假设

  example:
    问题: "设计用户认证系统"

    应用模板:
      1. 核心问题: 用户身份验证和授权
      2. 约束: 安全性、性能、可扩展性
      3. 目标: 安全、快速、易用
      4. 假设: 已有用户数据库
```

### 2. 推理模板 (Reasoning Template)

**用途**：系统化推理和分析

```yaml
reasoning_template:
  structure:
    1. 陈述事实
    2. 识别关系
    3. 推导结论
    4. 验证逻辑

  example:
    问题: "为什么登录慢？"

    应用模板:
      1. 事实: 登录需要3秒，预期<1秒
      2. 关系: 登录涉及数据库查询和密码验证
      3. 推导: 可能是数据库查询慢或密码算法复杂
      4. 验证: 检查数据库日志和密码算法
```

### 3. 验证模板 (Verification Template)

**用途**：全面验证结果质量

```yaml
verification_template:
  structure:
    1. 功能验证
    2. 性能验证
    3. 安全验证
    4. 边界验证

  example:
    功能: 登录系统

    应用模板:
      1. 功能: 能否成功登录？
      2. 性能: 响应时间是否<1秒？
      3. 安全: 密码是否加密？
      4. 边界: 错误密码如何处理？
```

## 模板使用流程

```yaml
usage_workflow:
  1. 选择模板:
     - 识别任务类型
     - 选择匹配模板

  2. 适配模板:
     - 根据具体场景调整
     - 添加特定细节

  3. 应用模板:
     - 按照模板结构处理
     - 确保完整性

  4. 优化模板:
     - 记录使用效果
     - 改进模板设计
```
