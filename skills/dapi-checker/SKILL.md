# API接口检查技能

## 工作流程

1. **接口分析**: 分析API接口设计和实现
2. **规范验证**: 检查是否符合API设计规范
3. **功能测试**: 验证接口功能完整性
4. **文档评估**: 评估API文档质量

## 关键决策点

- 接口设计规范
- 数据格式标准
- 错误处理机制
- 安全性检查

## 基本使用方法

提供API接口代码或文档，我将为您提供：
- 接口设计分析
- 规范符合性检查
- 改进建议
- 测试用例设计

## 常见模式

- REST API检查
- GraphQL接口验证
- 微服务接口分析
- API文档评估
