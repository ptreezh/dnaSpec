# DNASPEC Context Optimization - 基础应用

## 五大优化策略

### 1. 提升清晰度

```yaml
clarity_optimization:
  简化语言:
    - 使用更简单的词汇
    - 避免模糊表述
    - 添加必要解释

  改进结构:
    - 使用清晰的层次结构
    - 添加标题和分隔
    - 使用列表组织内容

  示例:
    before: "实现一个智能系统来处理数据"
    after: "构建一个机器学习分类器，处理用户行为数据"
```

### 2. 提升相关性

```yaml
relevance_optimization:
  删除无关内容:
    - 移除不相关的信息
    - 聚焦核心主题
    - 提升信息密度

  示例:
    before: 包含大量数据库设计细节（与认证无关）
    after: 只保留认证相关的核心信息
```

### 3. 提升完整性

```yaml
completeness_optimization:
  补充缺失信息:
    - 识别信息缺口
    - 补充关键细节
    - 添加边界情况

  示例:
    before: "用户系统包括注册、登录"
    after: "包括注册、登录、密码重置、账号管理"
```

### 4. 提升一致性

```yaml
consistency_optimization:
  统一术语:
    - 建立术语表
    - 统一概念表述
    - 避免同义词混用

  统一风格:
    - 统一命名风格
    - 统一格式规范
    - 统一表达方式

  示例:
    before: "用户"和"账号"混用
    after: 统一使用"用户"
```

### 5. 提升效率

```yaml
efficiency_optimization:
  删除冗余:
    - 删除重复内容
    - 合并相似信息
    - 简化冗长表述

  提升密度:
    - 使用更简洁的表达
    - 提高信息密度
    - 减少字数

  示例:
    before: "用户可以进行登录操作，登录后可以访问系统"
    after: "用户登录后可访问系统"
```
