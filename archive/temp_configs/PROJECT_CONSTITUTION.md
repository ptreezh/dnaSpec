# DNASPEC Context Engineering Skills System - 项目宪法 (Project Constitution)

## 1. 项目核心使命

DNASPEC (Dynamic Specification-Guided Skills System) Context Engineering Skills System 是一个**AI原生的上下文工程增强工具集**，专门设计用于AI CLI平台，利用AI模型的原生智能提供专业的上下文分析、优化和结构化能力。

## 2. 核心哲学

### 2.1 第一性原理
- **AI原生智能**: 100%利用AI模型原生智能，无本地模型依赖
- **指令工程**: 通过高质量指令模板引导AI模型执行专业任务
- **专业增强**: 为AI CLI平台提供专业化上下文工程能力
- **平台集成**: 与AI CLI平台无缝集成，作为增强工具

### 2.2 设计哲学
- **不替代AI**: 充分利用AI模型已有原生能力，而非构建本地模型
- **指令驱动**: 通过精确提示工程实现专业功能
- **模块化设计**: 独立技能模块，便于扩展和维护
- **API优先**: 通过AI API实现功能，而非本地算法

### 3.2 关键组件

#### 3.2.1 DNASPEC Skills (核心技能)
- ContextAnalysisSkill: 利用AI原生智能进行五维质量分析
- ContextOptimizationSkill: 利用AI原生推理和生成能力进行优化
- CognitiveTemplateSkill: 利用AI原生思维能力应用认知模板

#### 3.2.2 DNASPEC Framework (DSGS框架集成)
- 继承自DSGSSkill基类
- 实现统一的execute接口
- 提供标准错误处理和结果格式

#### 3.2.3 Platform Integration (平台集成)
- CLI命令集成
- API接口兼容
- 结果格式化输出

## 4. 功能定位

### 4.1 核心技能集
1. **Context Analysis Skill**: 五维指标分析（清晰度、相关性、完整性、一致性、效率）
2. **Context Optimization Skill**: 基于AI推理的多目标优化
3. **Cognitive Template Skill**: 五种认知模板应用（思维链、少样本、验证等）

### 4.2 交互模式
- **CLI模式**: 直接在AI CLI中使用
- **函数调用模式**: 在Python代码中调用
- **API模式**: 通过API接口集成

### 4.3 平台支持
- Claude CLI (通过指令调用)
- Gemini CLI (通过API调用)
- Qwen CLI (通过指令调用)
- 其他AI CLI平台

## 5. 技术约束

### 5.1 AI原生原则
1. **零本地模型依赖**: 100%使用AI模型原生能力
2. **指令驱动**: 通过高质量提示引导AI模型
3. **结果结构化**: 将AI非结构化响应转为结构化结果
4. **接口标准化**: 遵循DSGS统一接口规范

### 5.2 性能目标
- 指令构造 < 50ms
- 响应解析 < 100ms
- 总体响应时间 < AI模型API响应时间
- 保持AI模型原生能力

### 5.3 质量标准
- 指令质量 > 95% (引导AI准确完成任务)
- 结果结构化 > 90% (AI响应正确解析)
- API兼容性 100%
- 向后兼容性 > 95%
