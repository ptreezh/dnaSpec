AI辅助开发的强大工具集

DSGS Context Engineering Skills提供专业的上下文分析、优化和认知模板应用功能，支持多语言AI助手平台

核心功能

上下文分析

分析文本质量的五个维度：清晰度、相关性、完整性、一致性和效率

支持标准和增强两种模式

认知模板

应用认知模板到任务，提升推理质量和结果准确性

包括思维链、少示例学习、验证检查等框架

上下文优化

基于分析结果优化文本内容，提升质量和可用性

支持Token预算优化和记忆集成考量

AI安全工作流

临时工作区管理系统防止AI生成的临时文件污染主项目

当临时文件超过20个时自动提醒整理

Git集成

完整的Git工作流支持，包括worktree和CI/CD功能

支持分支管理和版本控制

跨平台支持

兼容Claude CLI、Gemini CLI、Qwen CLI等多种AI工具

统一的斜杠命令接口

安装指南

通过以下步骤快速安装DSGS Context Engineering Skills：

# 克隆项目
git clone https://github.com/ptreezh/dnaSpec.git
cd dnaSpec

# 安装本项目
pip install -e .

# 验证安装
python -c "from clean_skills.context_analysis import execute; print('DSGS安装成功!')"

使用方法

命令行使用

/speckit.dsgs.context-analysis "分析这个需求文档的质量" mode=enhanced
/speckit.dsgs.cognitive-template "如何提高性能" template=verification
/speckit.dsgs.context-optimization "优化需求描述" optimization_goals=clarity,relevance
/speckit.dsgs.architect "设计电商系统架构"
/speckit.dsgs.git-skill "operation=status"
/speckit.dsgs.temp-workspace "operation=create-workspace"

Python API

from clean_skills.context_analysis import execute as context_analysis_execute

# 标准模式
result = context_analysis_execute({
    'context': '设计用户登录功能',
    'mode': 'standard'
})

# 增强模式
result = context_analysis_execute({
    'context': '设计高安全性的用户登录功能', 
    'mode': 'enhanced'
})

AI安全工作流

AI生成内容遵循以下安全流程，防止项目被临时文件污染：

安全开发流程

  1. 生成阶段: AI输出首先存入临时工作区
  2. 整理阶段: 当临时文件超过20个时自动提醒
  3. 确认阶段: 人工验证后文件移至确认区域
  4. 提交阶段: 确认文件可安全提交到Git仓库
  5. 清理阶段: 自动清理临时工作区

安全使用示例

from clean_skills.temp_workspace_skill import execute as temp_execute

# 创建临时工作区（AI生成前必须）
temp_execute({'operation': 'create-workspace'})

# AI生成内容到临时区
temp_execute({
    'operation': 'add-file',
    'file_path': 'generated_code.py',
    'file_content': '# AI生成的代码'
})

# 验证后确认文件
temp_execute({'operation': 'confirm-file', 'confirm_file': 'generated_code.py'})

# 清理临时区
temp_execute({'operation': 'clean-workspace'})
