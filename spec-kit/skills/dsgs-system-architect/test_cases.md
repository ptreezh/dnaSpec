# DSGS System Architect技能TDD测试用例

## 测试1: 基础系统架构设计
**输入**: "为一个博客系统设计系统架构，需要前端展示、后端管理、数据库存储"
**预期输出**:
- 识别出前端、后端、数据库三层架构
- 推荐适当的技术栈 (如React前端, Node.js后端, MongoDB数据库)
- 设计API接口规范
- 考虑基本的安全措施

## 测试2: 微服务架构设计
**输入**: "为一个大型电商平台设计微服务架构，包含用户服务、产品服务、订单服务"
**预期输出**:
- 设计独立的微服务组件
- 定义服务间通信协议 (如REST API, gRPC, Message Queue)
- 推荐服务发现和负载均衡方案
- 考虑分布式数据管理策略

## 测试3: 高可用性架构
**输入**: "设计一个金融交易系统，要求99.99%可用性，高安全性"
**预期输出**:
- 设计冗余和故障转移机制
- 推荐多区域部署策略
- 考虑CAP定理的权衡
- 包含安全认证和授权机制

## 测试4: 高性能架构

## 测试5: 数据密集型架构
