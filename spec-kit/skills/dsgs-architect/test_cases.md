# DSGS Architect技能TDD测试用例

## 测试1: 基础架构分析
**输入**: "创建一个简单的电商系统，需要用户管理、产品管理、订单处理功能"
**预期输出**: 
- 识别出核心组件：用户服务、产品服务、订单服务
- 提出微服务架构
- 建议使用REST API进行服务通信
- 识别数据存储需求

## 测试2: 复杂架构设计
**输入**: "设计一个高并发的社交媒体平台，需要支持千万级用户，实时消息推送，内容推荐算法"
**预期输出**:
- 提出分布式架构
- 建议使用消息队列处理实时通信
- 推荐缓存策略处理高并发
- 提出数据分片方案
- 考虑CDN内容分发

## 测试3: 技术栈推荐
**输入**: "为一个金融交易系统设计架构，需要高安全性、高可用性、低延迟"
**预期输出**:
- 推荐安全的技术栈
- 提出高可用性设计模式
- 建议低延迟通信方案
- 包含安全认证授权机制
- 考虑审计和合规性

## 测试4: 依赖协调
**输入**: "设计一个数据处理流水线，包含数据采集、清洗、分析、可视化模块"
**预期输出**:
- 定义模块间接口
- 提出数据格式标准
- 建议错误处理机制
- 包含监控和日志方案

## 测试5: 可扩展性设计
**输入**: "为初创公司设计可扩展的SaaS架构，预计用户量从千级增长到百万级"
**预期输出**:
- 提出水平扩展方案
- 建议弹性资源分配
- 考虑成本优化策略
- 包含未来扩展路径

## 测试6: 安全设计
**输入**: "设计一个医疗健康应用架构，需要处理敏感健康数据"
**预期输出**:
- 提出端到端加密方案
- 建议访问控制机制
- 包含数据隐私保护措施
- 考虑合规性要求（如HIPAA）

## 测试7: 性能优化
**输入**: "为实时视频处理系统设计架构，需要处理4K视频流"
**预期输出**:
- 提出高性能计算方案
- 建议GPU/硬件加速
