DNASPEC项目使命

DNASPEC（DNA Specification Context System）是一个以生物DNA为灵感的平台，专注于系统从简单Idea到复杂智能化系统的全生命周期发展。

我们的核心使命是通过一系列精心设计的技能（skills），帮助用户在AI开发过程中自动搭建一套智能化自动增长的规范化系统，遵循认知过程的格式塔原则，实现从简单到复杂的一般性发展演化。

未来每个系统都将是综合集成研讨厅，融合定性与定量两种处理方式，定性部分以思维链和提示词优化为主，以置信度审核和多源多维度思考的共识计算为判断准则；定量部分则通过明确的、可固化的逻辑和算法，以代码方式实现。

核心设计原则

格式塔认知原则

系统各部分应形成统一、连贯的整体，遵循接近性、相似性、连续性、闭合性等格式塔原则，确保用户认知的一致性。

自顶向下分解

从整体概念出发，逐步分解为具体的、可执行的功能模块，确保分解过程中的逻辑一致性。

上下文隔离

通过智能体和模块化实现上下文隔离，避免单一上下文无限膨胀/腐化，保持系统效率。

渐进式增长

系统应能够从简单到复杂平稳增长，每一步增长都建立在前一步的坚实基础上。

自动化验证

通过自动化工具持续验证系统的一致性、完整性，防止任务不对齐和逻辑不一致。

智能与确定性平衡

在系统智能化和可预测性之间保持平衡，确定性逻辑用代码，不确定性用智能体处理。

技能框架与功能

DNASPEC技能系统围绕综合集成研讨厅理念设计，确保定性定量处理方式有机结合，实现智能化与自动化的平衡。每个技能都旨在解决AI开发中的特定挑战，提供结构化、一致性的解决方案。

上下文分析技能 (/dnaspec.context-analysis)

分析输入上下文质量，从清晰度、相关性、完整性、一致性、效率五个维度进行评估，是确保系统演进过程中质量一致的核心技能。

  • 清晰度：需求表达是否明确无歧义
  • 相关性：需求与项目目标的关联度
  • 完整性：需求是否包含所有必要信息
  • 一致性：需求内部是否逻辑自洽
  • 效率：需求描述是否简洁有效

全生命周期应用：从Idea阶段到智能体集成阶段，持续确保上下文质量。

上下文优化技能 (/dnaspec.context-optimization)

使用AI驱动的改进来优化上下文质量，避免上下文膨胀和概念模糊，确保上下文保持在适当复杂度。

  • 简化模糊需求，提高清晰度
  • 补充缺失信息，增强完整性
  • 统一不一致表达，确保一致性
  • 精简冗余描述，提升效率
  • 标准化描述格式，保证相关性

防爆炸机制：限制上下文长度和复杂度，防止失控增长。

认知模板技能 (/dnaspec.cognitive-template)

应用认知框架（链式思维、少样本学习、验证、角色扮演）来结构化复杂任务，遵循格式塔认知原则。

  • Chain of Thought：逐步推理分析复杂问题
  • Few Shot：从示例中学习执行任务模式
  • Verification：双重检查结果的准确性
  • Role Playing：扮演特定角色进行专业分析

自顶向下分解：将复杂需求分解为可管理的子任务。

系统架构师技能 (/dnaspec.architect)

设计系统架构和组件，支持微服务、单体等多种架构风格，防止架构失控和扩展性问题。

  • 微服务架构：支持模块化和独立部署
  • 单体架构：适合小型项目快速开发
  • 事件驱动：支持异步处理和扩展
  • 分层架构：确保职责分离和可维护性

可扩展设计：考虑从简单到复杂的演进路径。

智能体创建技能 (/dnaspec.agent-creator)

根据局部上下文创建专门的智能体，执行特定任务，避免整个系统上下文膨胀，实现局部智能体架构。

  • 专业化分工：创建不同领域的专用智能体
  • 上下文隔离：每个智能体只关注自己的领域
  • 任务分配：将复杂任务分发给相应的智能体
  • 协同工作：多个智能体协作完成复杂任务

防膨胀机制：通过智能体实现功能模块化。

模块化技能 (/dnaspec.modulizer)

将系统分解为可重用模块，实现自底向上模块化，避免系统过于庞杂，确保模块间边界清晰。

  • 组件识别：自动识别系统中的可复用组件
  • 模块设计：规划模块结构和接口定义
  • 依赖分析：识别模块间的依赖关系
  • 接口标准化：确保模块间接口一致

解耦合设计：确保各模块独立发展，降低系统复杂性。

工作区管理技能 (/dnaspec.workspace)

在安全工作区中管理AI生成的文件，提供隔离环境，防止AI生成文件污染主项目。

  • 文件隔离：AI生成文件在独立环境处理
  • 安全操作：防止意外修改主项目文件
  • 版本控制：独立管理AI生产的代码版本
  • 质量检查：在整合前验证文件质量

安全开发：确保AI辅助开发过程的可控性。

Git操作技能 (/dnaspec.git)

安全地执行Git工作流操作，支持AI辅助开发的版本管理需求。

  • 状态检查：查看仓库状态和变更
  • 提交管理：安全地提交和推送更改
  • 分支操作：管理功能开发分支
  • 历史记录：查看和管理提交历史

版本保护：防止AI误操作导致代码丢失。

任务分解技能 (/dnaspec.task-decomposer)

将复杂项目分解为遵循KISS、YAGNI、SOLID原则的原子任务，确保任务分解的简洁性和有效性。

  • KISS原则：保持任务简单清晰
  • YAGNI原则：只分解当前需要的功能
  • SOLID原则：确保任务职责单一
  • 隔离工作区：为每个任务创建独立环境

防爆炸机制：限制任务数量和复杂度。

约束生成技能 (/dnaspec.constraint-generator)

基于初始宪法需求生成动态约束，实现需求版本管理和时间点恢复机制，确保后续需求与初始目标对齐。

  • 宪法锁定：将初始需求设为系统宪法
  • 对齐检查：确保新需求与宪法一致
  • 冲突检测：识别需求间冲突
  • 时间恢复：支持时间点回滚和重构

一致性保障：维护需求与目标的一致性。

API检查技能 (/dnaspec.api-checker)

实现模块级、子系统级、系统级的分级审核，验证不同层级间的调用对齐和作用域管理，确保各层级接口一致性。

  • 分级审核：按模块、子系统、系统层级检查
  • 接口对齐：验证不同层级间接口一致性
  • 作用域管理：确保全局到局部的作用域边界
  • 模块分解：检查按模块分解的独立接口定义

多层级对齐：确保各层级间的协调一致。

系统全生命周期演进

从简单想法到复杂智能系统的发展过程中，DNASPEC技能系统支持每个阶段的质量保障和增长引导。

阶段1：Idea孵化阶段

核心任务：将模糊概念转化为清晰需求，应用格式塔认知原则，形成完整概念框架。

挑战：概念模糊、需求不完整、用户意图不明确

技能应用：

  • Idea阶段 使用 /dnaspec.context-analysis 分析概念清晰度
  • Idea阶段 使用 /dnaspec.context-optimization 明确概念表达
  • Idea阶段 使用 /dnaspec.cognitive-template 通过链式思维结构化概念
示例需求:
/dnaspec.context-analysis "创建一个能帮我管理时间和任务的应用"
/dnaspec.context-optimization "时间管理应用基础需求" --goals clarity,completeness

阶段2：需求规范化阶段

核心任务：将概念转化为规范化的功能需求，遵循自顶向下任务分解原则，将整体概念拆解为具体的、可管理的功能模块。

挑战：需求分解可能遗漏重要功能，模块间可能存在冲突或重复

技能应用：

  • 需求阶段 使用 /dnaspec.cognitive-template --template chain_of_thought 分解功能
  • 需求阶段 使用 /dnaspec.architect 设计微服务架构
  • 需求阶段 使用 /dnaspec.constraint-generator 锁定核心需求
需求分解示例:
/dnaspec.architect "时间管理应用" --style microservices
/dnaspec.constraint-generator "用户认证和数据安全核心需求"

阶段3：功能细化与模块化阶段

核心任务：逐步细化和扩展系统功能，关注模块独立性和接口一致性，避免上下文腐蚀和架构失控。

挑战：上下文膨胀/腐蚀，系统复杂度快速增长，模块间依赖混乱

技能应用：

  • 细化阶段 使用 /dnaspec.modulizer 实现系统模块化
  • 细化阶段 使用 /dnaspec.api-checker 验证模块接口一致性
  • 细化阶段 使用 /dnaspec.workspace 管理独立开发环境
模块化示例:
/dnaspec.modulizer "用户管理、任务管理、通知管理模块"
/dnaspec.api-checker "验证用户服务与任务服务接口对齐"

阶段4：智能化集成阶段

核心任务：引入智能体(agents)提升系统智能化水平，通过局部智能体处理复杂意图，避免全局上下文膨胀。

挑战：全面测试复杂，任务对齐困难，智能体间逻辑不一致

技能应用：

  • 智能阶段 使用 /dnaspec.agent-creator 创建专业智能体
  • 智能阶段 使用 /dnaspec.context-analysis 验证智能体集成度
  • 智能阶段 使用 /dnaspec.git 进行版本控制
智能体集成示例:
/dnaspec.agent-creator "时间管理专家，帮助用户优化日程安排和任务优先级"
/dnaspec.agent-creator "数据洞察专员，分析用户习惯并提供建议"

未来综合集成研讨厅理念

我们的愿景是每个系统都成为综合集成研讨厅，融合定性定量处理方式，实现智能化与自动化的平衡。

综合集成研讨厅是一个集成了定性与定量分析能力的平台，定性部分处理用户意图的不确定性、创造性需求，通过思维链、提示词优化、置信度审核和多源多维度思考的共识计算进行判断；定量部分处理确定性逻辑、算法运算，通过固化代码实现可预测、高效的功能。

系统遵循认知过程的格式塔原则，确保用户在交互过程中感知到统一、连贯的智能体体验。局部智能体架构实现上下文隔离，防止全局上下文膨胀，同时通过分级审核机制确保模块、子系统、系统级的接口对齐和作用域管理。

这种理念将确定性逻辑用代码和算法固化，将不确定性分析通过智能体处理，将需要用户决策的模糊问题以最直观的决策问答形式呈现给用户界面，实现真正智能化的自动增长系统。

综合集成方法

定性处理机制

采用认知框架处理复杂、模糊或不确定性问题，通过多轮推理和逐步分析，结合置信度审核对推理过程验证确认，通过多源多维度思考的共识计算提高判断准确度。

应用于用户意图理解、需求澄清、创新性设计、复杂问题分析等场景。

定量处理机制

采用算法和逻辑处理明确、可固化的计算任务，具有确定性输入输出关系，用代码精确实现。通过固化定量逻辑，保证处理高效性和一致性，为定性推理提供可靠支撑。

应用于数据计算、业务规则验证、性能优化、安全检查等场景。

智能体社会架构

不同粒度的智能体形成社会组织结构，协同完成复杂任务。模块级智能体负责具体功能，子系统级智能体管理功能域，系统级智能体整体协调，任务级智能体执行原子任务。

每个智能体专注当前任务上下文，实现上下文隔离，避免全局上下文膨胀，形成专业分工协作体系。

分级审核与对齐

实现模块级、子系统级、系统级多层级审核架构，验证各层级间接口对齐和作用域管理。确保模块独立开发、子系统集成、系统整体一致性。

通过多级审核确保系统从局部到整体的一致性和质量。

动态约束管理

基于初始宪法需求生成动态约束，确保后续需求与初始目标对齐。通过版本管理、时间点恢复机制，防止需求偏离初心，支持变更追溯。

维护系统演进过程中的目标一致性，避免需求碎片化和偏离。

格式塔认知完整性

遵循接近性、相似性、连续性、闭合性等认知原则，确保系统各部分形成统一连贯整体。确保用户交互时感知到一致统一的智能体体验。

维持系统认知完整性，避免逻辑不一致和用户体验碎片化。
