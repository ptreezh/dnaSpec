# spec.kit 使用场景示例

## 概述

本文档提供spec.kit工具包在不同使用场景下的实际应用示例，帮助用户理解如何在真实项目中使用各种技能。请注意，目前所有技能都以完整命令形式使用（如 /speckit.specify），快捷方式尚不可用。

## 场景1: 新项目启动

### 业务需求
启动一个新的电商平台项目，需要建立完整的开发规范和架构设计。

### 使用步骤

#### 1. 建立项目宪法
```
/speckit.constitution 电商项目
```
**作用**: 建立项目的编码标准、开发规范和团队协作原则

#### 2. 需求规格化
```
/speckit.specify 用户需要能够浏览商品、下单购买、支付、查看订单状态
```
**作用**: 将业务需求转换为详细的技术规格

#### 3. 技术规划
```
/speckit.plan 电商平台的技术实现
```
**作用**: 确定技术栈、架构决策和开发方法

#### 4. 任务分解
```
/speckit.tasks 电商平台开发计划
```
**作用**: 将项目分解为具体的开发任务

## 场景2: 内容质量优化

### 业务需求
优化一份技术文档，使其更清晰、完整、易于理解。

### 使用步骤

#### 1. 内容分析
```
/context-analysis 这是需要优化的技术文档内容...
```
**作用**: 分析文档的清晰度、相关性、完整性等质量指标

#### 2. 内容优化
```
/context-optimization 这是需要分析的文档内容...
```
**作用**: 基于特定目标优化文档质量

#### 3. 完整工作流
```
/context-engineering-workflow 需要全面改进的文档内容...
```
**作用**: 一次性完成分析→优化→认知增强的完整流程

## 场景3: 复杂系统架构设计

### 业务需求
设计一个支持千万级用户的社交媒体平台架构。

### 使用步骤

#### 1. 架构设计
```
/dnaspec-architect 社交媒体平台需求：支持千万级用户，实时消息推送，内容推荐
```
**作用**: 设计分层架构，协调各项子技能

#### 2. 任务分解
```
/dnaspec-task-decomposer 社交媒体平台架构
```
**作用**: 将复杂架构分解为实施任务

#### 3. 智能体创建
```
/dnaspec-agent-creator 消息推送服务需求
```
**作用**: 创建智能体处理实时消息推送

#### 4. 约束生成
```
/dnaspec-constraint-generator 实时系统架构
```
**作用**: 生成性能和可靠性约束

## 场景4: API文档质量检查

### 业务需求
检查多个微服务之间的API接口一致性。

### 使用步骤

#### 1. 接口检查
```
/dnaspec-dapi-checker 消息服务、用户服务、内容服务的API文档
```
**作用**: 检查接口一致性，识别不匹配问题

#### 2. 质量分析
```
/context-analysis-enhanced API文档内容
```
**作用**: 使用增强分析检查API文档质量

## 场景5: 模块化重构

### 业务需求
将一个单体应用的部分模块进行重构和封装。

### 使用步骤

#### 1. 模块成熟度评估
```
/dnaspec-modulizer 用户管理模块代码
```
**作用**: 评估模块的成熟度和封装准备情况

#### 2. 模块化验证
```
/dnaspec-modulizer 订单处理模块代码
```
**作用**: 检查模块的内聚性和耦合度

## 高级使用模式

### 1. 工作流编排
```
# 完整的规格驱动开发流程
/speckit.constitution 项目类型
/speckit.specify 需求描述
/speckit.plan 规格内容
/speckit.tasks 计划内容
# 然后执行任务
```

### 2. 质量保证流程
```
# 内容质量保证流程
/context-analysis 初始文档
/context-optimization 优化后的文档
/context-analysis-enhanced 进一步优化文档
```

### 3. 架构验证流程
```
# 架构设计验证流程
/dnaspec-architect 需求
/dnaspec-task-decomposer 架构
/dnaspec-constraint-generator 架构
/dnaspec-dapi-checker 服务接口
```

## 最佳实践

### 1. 新手引导
- 从 `/speckit.constitution` 开始建立项目规范
- 使用 `/speckit.specify` → `/speckit.plan` → `/speckit.tasks` 的线性流程
- 熟悉后尝试并行使用多个技能

### 2. 高效工作
- 使用增强版技能（如 `/context-analysis-enhanced`）获得更专业的结果
- 对于复杂文档，使用 `/context-engineering-workflow` 一次性解决问题
- 利用Python脚本增强的技能获取更精确的结果

### 3. 团队协作
- 通过 `/speckit.constitution` 统一团队规范
- 使用 `/speckit.specify` 确保需求理解一致
- 使用 `/speckit.tasks` 分解工作便于分工
